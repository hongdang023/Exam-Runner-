# DAC FRAMEWORK - MASTER TEMPLATE
## (Outcome-Based Project Management)

Đây là **Bộ khung mẫu (Master Template)** được chuẩn hóa từ logic của dự án Conan Platform, dùng để áp dụng cho việc tổ chức thông tin và quản trị mọi dự án dựa trên kết quả (Outcome-based) trong tương lai.

## 1. TRIẾT LÝ CỐT LÕI (CORE LOGIC)
Mọi dự án áp dụng Framework này đều phải dịch chuyển từ việc "liệt kê tính năng" sang "quản trị kết quả đầu ra".
*   **Hành động**: Luôn có bằng chứng (Proof of Work).
*   **Tiến độ**: Đo lường bằng Milestone đạt được, không đo bằng thời gian trôi qua.
*   **Phân cấp**: Tuân thủ nghiêm ngặt 3 Phase SDLC (Requirement -> Design -> Implementation).

## 2. HƯỚNG DẪN SỬ DỤNG (HOW TO USE)
1.  **Clone**: Copy toàn bộ folder `DAC/` vào thư mục dự án mới của bạn.
2.  **Khởi tạo**: Đọc kỹ file `RULES.md` để hiểu cách đặt tên và tổ chức.
3.  **Tùy chỉnh A0**: Mở `A_REQUIREMENTS/A0_Standards.md` và định nghĩa "Bộ tiêu chuẩn kết quả" riêng cho dự án mới.
4.  **Phát triển**: Điền nội dung vào các file README trong từng Phase để dẫn dắt đội ngũ.

---
*Tài liệu này được trích xuất và chuẩn hóa từ hệ thống quản trị dự án Conan Platform.*
*   Ví dụ: Trước khi viết Usecase, phải đọc `Requirement_Standards`.

### Quy tắc 3: Cấu trúc file Markdown
*   Sử dụng Prefix số thứ tự (A0, A1, B1...) để thể hiện luồng đọc ưu tiên.
